# Tower Defence + FPS-герой — дизайн

**Дата:** 2026-06-19
**Статус:** утверждён, готов к плану реализации

## 1. Концепция

Классический tower defence в полном 3D в браузере. Враги идут по фиксированному
пути от точки спавна к базе. Игрок ставит башни сверху (top-down). В любой момент
по клавише **Tab** игрок «вселяется» в героя у финиша — бегает от первого лица,
целится мышью, помогает отстреливать врагов. Можно играть и пассивно: смотреть
сверху, как башни защищают базу. Цель — провести базу через все волны.

Ключевая фишка: **одна 3D-симуляция, две точки зрения**. Враги и башни не знают
про камеру; камера — только точка обзора. Это держит гибрид дешёвым в реализации.

## 2. Стек и платформа

- **Браузер**, полный 3D.
- **Babylon.js** + **TypeScript**, сборка **Vite**.
- Геометрия — примитивы / low-poly прямо в коде (капсулы, цилиндры, боксы).
  Готовые/свои модели — вне MVP, фокус на механике.
- Одна сцена, **две камеры**:
  - top-down (орто/ArcRotate сверху) — режим строительства;
  - FPS (`UniversalCamera` + pointer-lock) — режим героя.
  - Tab переключает активную камеру; мир один и тот же.
- Физика лёгкая: коллизии Babylon (`moveWithCollisions` / camera collisions) для
  героя, **raycast** для стрельбы. Полноценный Havok — вне MVP.

## 3. Архитектура

Каждый модуль — одна задача, общение через явные интерфейсы и `EventBus`.

```
src/
  main.ts            — bootstrap, игровой цикл, рендер
  core/
    GameState.ts     — золото, жизни базы, номер волны, фаза (build/wave/gameover)
    EventBus.ts      — события (enemyKilled, baseHit, waveCleared, ...)
  world/
    Level.ts         — сетка карты, путь (waypoints), точки строительства
    PathFollower.ts  — движение по waypoints (общий механизм)
  towers/
    Tower.ts         — база: range, fireRate, damage, выбор цели
    TowerTypes.ts    — данные cannon / slow / sniper
    TowerManager.ts  — постройка, апгрейд, продажа, валидация клеток
    Projectile.ts    — снаряд / хитскан, нанесение урона
  enemies/
    Enemy.ts         — HP, speed, движение по пути, смерть -> золото
    EnemyTypes.ts    — данные normal / fast / tank
    WaveManager.ts   — расписание волн, спавн, проверка «волна зачищена»
  hero/
    HeroController.ts— FPS-движение (WASD), коллизии
    HeroWeapon.ts    — стрельба raycast, урон, cooldown/перезарядка
    HeroState.ts     — HP героя, смерть, респавн-таймер
  camera/
    CameraRig.ts     — top-down <-> FPS, pointer-lock, Tab-переключение
  ui/
    HUD.ts           — золото, жизни, волна, HP героя, прицел
    BuildMenu.ts     — выбор и постановка башни (top-down режим)
```

**Принцип изоляции:** враги и башни не знают про камеру и про героя как про
«игрока» — герой для них обычный источник урона. Симуляция всегда реалтайм и
одинакова независимо от выбранной камеры.

## 4. Поток данных / игровой цикл

Единый тик в `scene.onBeforeRender`:

1. `WaveManager` спавнит врагов по расписанию.
2. `Enemy` двигаются через `PathFollower` по waypoints.
3. `Tower` ищут цель в радиусе, стреляют `Projectile`.
4. Попадание снижает HP врага; смерть -> золото в `GameState` + событие
   `enemyKilled`.
5. Герой тикает в том же цикле: ввод -> `HeroController` двигает с коллизиями;
   ЛКМ -> `HeroWeapon` делает raycast по врагам, наносит урон.
6. Враг дошёл до базы -> базе −1 жизнь, враг удаляется (событие `baseHit`).
7. Жизни базы = 0 -> gameover. Все волны зачищены -> victory.

## 5. Фазы игры

- **Build** (между волнами): время на постройку/апгрейд сверху; кнопка «старт
  волны».
- **Wave**: враги идут; можно одновременно строить и прыгать в героя. Tab доступен
  всегда.
- **GameOver / Victory**: экран результата, рестарт.

## 6. Управление

- **Tab** — переключение top-down <-> герой.
- **Top-down:** ЛКМ — поставить выбранную башню на валидную клетку; выбор башни —
  цифры/меню; клик по башне — апгрейд/продажа.
- **Герой:** WASD — движение; мышь — обзор (pointer-lock); ЛКМ — стрельба;
  перезарядка — авто или R (опц.).

## 7. Контент MVP

**Башни:**
- `cannon` — сбалансированный урон/дальность/скорость.
- `slow` — замедляет врагов, малый/нулевой урон.
- `sniper` — редкий выстрел, большой урон, большая дальность.

**Враги:**
- `normal` — базовый.
- `fast` — мало HP, высокая скорость.
- `tank` — много HP, низкая скорость.

**Прогресс:**
- 10 волн нарастающей сложности; прошёл все -> победа.
- База ~20 жизней (тюнится).
- Экономика: золото за убийство + бонус за волну; тратится на постройку/апгрейд.
- Апгрейд башен: 3 уровня (рост урона и дальности).
- 1 карта-уровень с фиксированным путём.

**Герой:**
- Смертен; при смерти — респавн через таймер у базы.
- Только стрельба (без утилити-способностей в MVP).

## 8. Тестирование

Vitest для чистой логики без рендера:
- `PathFollower` — корректное движение по waypoints, достижение конца.
- `Tower` — выбор цели в радиусе, частота стрельбы.
- урон/смерть врага, начисление золота.
- `GameState` — экономика, жизни базы, смена фаз.
- `WaveManager` — спавн и определение «волна зачищена».
- `HeroWeapon` — расчёт урона по попаданию.

Рендер и Babylon-специфика — ручной прогон в браузере.

## 9. Вне MVP (YAGNI)

Мультиплеер, сохранения, звук/музыка, несколько карт, maze-pathfinding с A*,
утилити-способности героя, мобильное управление, качественные 3D-модели,
полноценная физика Havok. Всё — после рабочего MVP.
