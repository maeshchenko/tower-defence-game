# Polish P2 — Game Feel / Juice

**Дата:** 2026-06-20
**Статус:** черновик дизайна, ожидает ревью
**Программа:** «Polish & game feel», саб-проект 2 из 4 (после P1 rendering).

## 0. Контекст

P1 дал свет/тени/пост-обработку/траву. P2 добавляет **отзывчивость** — «сок»
(juice): экран реагирует на выстрелы, попадания и смерти, не меняя механику.
Источники: Vlambeer «Art of Screenshake» (сдержанность!), juice-чеклист
(particles/hitstop/tweening). Арт-дирекшен прежний: Stylized Toon.

Сейчас FX бедные: `spawnFlash` — расширяющаяся сфера (muzzle/impact), смерть —
только death-клип. Нет тряски, хитстопа, частиц, отдачи, трейлов.

## 1. Цель

Сделать каждое действие «вкусным», сохранив **читаемость** (TD-правило: при
куче врагов экран не должен превращаться в кашу):
- **Screenshake** (trauma-based, затухающий) — выстрел героя, попадание, смерть.
- **Hitstop** — короткая микро-пауза симуляции на КРУПНЫХ событиях.
- **Частицы (GPU)** вместо сфер-вспышек: muzzle / impact / death burst.
- **Отдача**: камера-кик при выстреле героя; откат ствола башни.
- **Hit-blink**: враг на попадании коротко вспыхивает белым (emissive).
- **Death-pop**: на смерти быстрый scale-pop + оседание/затухание трупа.
- **Трейлы** у снарядов.

**Вне P2:** UI/HUD, pause/скорость, звук, новый контент. Балом правит сдержанность:
эффекты слабые и быстрые.

## 2. Ключевые дизайн-решения (обоснованные)

1. **Hitstop только на крупном**, НЕ на каждом килле. При 20+ врагах фриз на
   каждом убийстве = заикание. Триггеры hitstop: смерть `tank`, добивание из
   `sniper`/базы, и удар по базе. Обычные киллы — только тряска+частицы. ~60–80мс.
2. **Тряска сильнее сверху, слабее в 3-м лице.** FPS-камеру трясти много =
   укачивает. Амплитуда в `hero`-режиме ≈ 0.4× от top-down.
3. **Симуляция и FX тикают по-разному.** Hitstop замораживает СИМ (враги/башни/
   снаряды-как-геймплей), но тряска/частицы/затухание идут по реальному времени —
   иначе фриз выглядит мёртвым.
4. **Trauma, не «толчок».** Накопление trauma∈[0,1], shake = trauma²·amp,
   затухание ~1.5/с. Несколько событий складываются, но trauma кэпится в 1.

## 3. Архитектура

Новый каталог `src/fx/`. Чистая математика — отдельно от Babylon, тестируется.

```
src/fx/
  Screenshake.ts  — ЧИСТО: trauma-контроллер. addTrauma(a), decay(dt),
                    intensity()=trauma². + offset(seed)->{x,y,roll}. Применение к
                    камере — тонкий метод applyTo(camera, dt, scale). Юнит-тест на
                    накопление/кэп/затухание.
  Hitstop.ts      — ЧИСТО: trigger(ms); update(realDtMs)->simScale (0 во фризе,
                    иначе 1). Юнит-тест на длительность/перекрытие триггеров.
  Particles.ts    — Babylon. burst(scene, kind, x,y,z, color): одноразовый
                    ParticleSystem (muzzle/impact/death), self-dispose по завершении.
  Trail.ts        — Babylon. attachTrail(scene, node, color, life): TrailMesh за
                    снарядом; dispose вместе со снарядом.
```

Правки существующих view (изолированно):
- `EnemyView`: `flashHit()` (короткий белый emissive-блик), усиленный `die()`
  (scale-pop, затем оседание+затухание трупа перед dispose).
- `TowerView`: `kickback()` — быстрый твин отката ствола назад и обратно.

## 4. Интеграция (main.ts, без смены механики)

Игровой цикл `onBeforeRenderObservable`:
```
realDt = getDeltaTime()/1000
simScale = hitstop.update(realDt*1000)     // 0 во фризе
dt = realDt * simScale                       // СИМ использует dt
... existing sim (enemies/towers/projectiles) on dt ...
shake.applyTo(activeCamera, realDt, scaleForMode)  // FX на realDt
updateFlashes(realDt); updateParticles? (PS自-tick)
```
Триггеры:
- `processHeroShot`: `shake.addTrauma(0.15)` + muzzle-частицы + (камера-кик через trauma).
- `fireTowerShot`: muzzle-частицы + `towerView.kickback()` (тряску НЕ копим — слишком часто).
- `applyHit` (попадание): impact-частицы (замена spawnFlash) + `enemyView.flashHit()` + `shake.addTrauma(0.08)`.
- `applyHit` (смерть): death-burst частицы + trauma (`tank` 0.4, иначе 0.15) +
  `hitstop.trigger(70)` если `tank` или добил `sniper`/база; `die()` с pop.
- `reachedBase` (урон базе): `shake.addTrauma(0.5)` + `hitstop.trigger(80)`.
- спавн снарядов (`spawnModelShot`/`spawnBall`): `attachTrail(...)` цвет по типу.

`spawnFlash`/`updateFlashes` (сферы) удаляются после переноса muzzle/impact на частицы.

## 5. Читаемость / производительность

- Частицы — небольшие пулы, короткий life, `disposeOnStop`. На `low`-пресете
  (из P1 Quality) можно резать count — но glow уже всегда вкл; держим частицы
  дешёвыми безусловно.
- Тряска кэпится; в `hero`-режиме скейл 0.4.
- Hitstop ≤80мс и только на крупном — не мешает контролю.
- Hit-blink ≤90мс — не залепляет силуэт врага.

## 6. Тестирование

- **Юнит (vitest):**
  - `Screenshake`: addTrauma складывается и кэпится в 1; intensity=trauma²;
    decay уводит в 0 за ожидаемое время; offset детерминирован от seed.
  - `Hitstop`: trigger держит simScale=0 заданные мс затем 1; повторный trigger
    во фризе продлевает до max(remaining, new); update без триггера = 1.
- **Babylon/визуал — браузер:** частицы muzzle/impact/death видны и читаемы;
  тряска ощутима сверху, мягче в 3-м лице, не тошнит; hitstop на смерти танка/
  ударе по базе заметен но не залипает; враг мигает на попадании; снаряды с
  трейлом; FPS стабилен в гуще волны; смена пресета P1 не ломает FX.

## 7. Риски

- **Тряска × pointer-lock (3-е лицо):** оффсет камеры не должен драться с
  mouse-look (CameraRig крутит alpha/beta). Применять shake как аддитивный
  оффсет позиции/`roll`, НЕ трогая alpha/beta цели.
- **Hitstop и анимации:** death-клип/локомоция управляются группами анимаций —
  фриз сим-`dt` не должен морозить FX-клипы (они тикают движком по реальному
  времени, ок) — проверить, что трупы не зависают.
- **Частицы-лики:** каждый burst обязан self-dispose; на `loadMap` живые PS/трейлы
  чистить вместе с остальным.
- **trauma в top vs hero:** при переключении камеры не дёргать — скейл по режиму.

## 8. Критерий готовности

Браузер: выстрелы/попадания/смерти дают тряску+частицы+отдачу; крупные события —
хитстоп; враги мигают и эффектно умирают; снаряды с трейлом; читаемость и FPS в
волне сохранены; `npm run typecheck`, `npm test`, `npm run build` зелёные;
механика и P1-визуал не сломаны.
