# Polish P1 — Rendering & Lighting (Stylized Toon)

**Дата:** 2026-06-19
**Статус:** черновик дизайна, ожидает ревью
**Программа:** «Polish & game feel» — первый из 4 саб-проектов.

## 0. Контекст программы

Игра — рабочий вертикальный срез (TD + FPS-герой гибрид, ~1700 LOC, Babylon.js 9).
Цель программы — довести «черновик» до ощущения настоящей игры по оси **polish &
game feel**. Направление выбрано пользователем; конкретику выбираю я на основе
ресёрча про профессиональные TD (Kingdom Rush / Bloons / Defender's Quest) и про
game juice (Vlambeer «Art of Screenshake»).

**Арт-дирекшен (утверждён): Stylized Toon** — опираемся на существующие
Kenney/KayKit ассеты (мультяшный low-poly). Никакой переделки моделей: насыщенная
палитра, мягкие тени, свечение emissive, туман для глубины. Цель — Kingdom Rush
vibe, но в 3D.

**Программа из 4 саб-проектов (каждый — свой spec → план):**
- **P1 — Rendering & lighting** ← этот документ (самый большой скачок «черновик→игра»)
- **P2 — Game feel / juice** (screenshake, hitstop, частицы, recoil, death-pop)
- **P4 — UI/HUD & TD-читаемость** (pause/скорость, total info transparency, settings)
- **P3 — Audio** (реальные сэмплы, музыка, spatial, микс) — последним, независим

Этот документ описывает **только P1**.

## 1. Цель P1

Заменить «плоскую» картинку (один `HemisphericLight`, flat `StandardMaterial`, без
теней, без пост-обработки) на согласованный стилизованный 3D-вид:

- Направленное солнце + **мягкие тени** → объекты «стоят на земле».
- Заполняющий ambient (остаётся hemi) → тени не чёрные.
- **Пост-обработка**: FXAA, bloom, ACES-тонмаппинг, экспозиция/контраст (цветокор).
- **GlowLayer** на emissive (снаряды, вспышки, база-кристалл, лучи врагов).
- **SSAO2** (контактное затенение, только на high-пресете).
- **Небо** (градиент) + лёгкий экспоненциальный **туман** для глубины.
- Чуть улучшенный материал земли (палитра/текстура вместо ровного зелёного).
- **Пресеты качества** (low/med/high) — пост-фх и тени дорогие, нельзя ронять
  слабые машины.

**Вне P1:** частицы/тряска/хитстоп (P2), редизайн HUD и меню настроек (P4), звук
(P3), переделка самих GLB-моделей, PBR-материалы, кастомные шейдеры/тун-рампы
(YAGNI — тонмаппинг + свет + тени дают нужный вид дешевле).

## 2. Принцип читаемости (TD best-practice)

Полировка не должна вредить ясности (Defender's Quest: «focus & thinking»). Тени и
bloom — умеренные: враги/башни остаются легко различимы, hp-бары и цифры урона
читаемы. Туман — тонкий, не скрывает игровое поле (вся карта видна без скролла,
это сохраняется). При сомнении — читаемость важнее красоты.

## 3. Архитектура

Два новых изолированных модуля; `main.ts` только подключает их и регистрирует
меши. Симуляция (враги/башни/герой) не меняется вообще.

```
src/rendering/
  Quality.ts       — ЧИСТЫЙ: QualityPreset = 'low'|'med'|'high';
                     resolveQuality(preset): QualityConfig (флаги: shadows, shadowMapSize,
                     ssao, bloom, fxaa, fog). Загрузка/сохранение пресета в localStorage.
                     Полностью юнит-тестируется (без Babylon).
  Environment.ts   — Babylon-обвязка. Создаёт: DirectionalLight (солнце) + остаётся
                     HemisphericLight (fill), ShadowGenerator, GlowLayer, опц. SSAO2,
                     skybox/градиент, fog, DefaultRenderingPipeline. Применяет
                     QualityConfig. API:
                       new Environment(scene, sun-fill refs, initial QualityConfig)
                       addShadowCaster(node: TransformNode)  — рекурсивно по child-мешам
                       setReceiver(mesh)                      — земля принимает тени
                       applyQuality(cfg)                      — пере-настройка на лету
                       dispose()
```

**Изоляция:** `Quality.ts` не знает про Babylon (чистая логика, тестируема).
`Environment.ts` не знает про геймплей — ему дают узлы, он вешает их в shadow map.
`main.ts` — единственное место, где они встречаются.

## 4. Тени — кто кастует, кто принимает

`ShadowGenerator` от направленного солнца. Регистрация кастеров — там же, где
создаются меши (динамически по ходу игры):

- **Земля** `ground` → `receiveShadows = true` (единственный приёмник; дороже не надо).
- **Кастеры** (через `env.addShadowCaster`):
  - герой — в `makeHero()`
  - враги — в конструкторе `EnemyView` (прокинуть callback/ссылку на env)
  - башни — в конструкторе `TowerView`
  - база keep, дорожные тайлы (опц.), solid-пропы (стены/камни/ящики/деревья) — в
    `buildEnvironment` / `buildProp`
- Снаряды/вспышки/частицы — **не** кастеры (мелкие, дорого, не нужно).

`addShadowCaster` рекурсивно добавляет все child-меши `TransformNode` в
`shadowGenerator.addShadowCaster`. На `low`-пресете тени выключены — метод
становится no-op.

## 5. Пресеты качества

| Параметр              | low      | med            | high              |
|-----------------------|----------|----------------|-------------------|
| FXAA                  | да       | да             | да                |
| bloom                 | нет      | да (умеренный) | да                |
| тени                  | нет      | да             | да                |
| shadow map size       | —        | 1024           | 2048              |
| SSAO2                  | нет      | нет            | да                |
| glow layer            | да       | да             | да                |
| fog                   | да       | да             | да                |
| тонмаппинг/экспозиция | да       | да             | да                |

Дефолт — **med**. Пресет хранится в `localStorage` (`td.quality`). Полноценный
селектор в меню настроек — в **P4**; в P1 — минимальный способ переключения
(клавиша циклит low→med→high + временный текстовый индикатор, или `?quality=high`
в URL). Это временно и будет заменено настройками в P4.

## 6. Цвет и свет (toon-настройка)

- **Солнце**: тёплый цвет (≈ `#fff0d0`), угол сверху-сбоку для выраженных, но не
  длинных теней. Intensity подбирается так, чтобы с hemi-fill сумма не пересвечивала.
- **Hemi-fill**: холодноватый (небо голубое, земля зелёная — `groundColor`),
  низкая intensity — только чтобы тени не были чёрными.
- **Image processing** (через pipeline): `toneMappingEnabled` (ACES),
  `exposure`/`contrast` для сочной, чуть контрастной палитры. HDR включён для
  корректного bloom.
- **Bloom**: невысокий threshold/scale — светятся только emissive-вещи (снаряды,
  вспышки, кристалл базы), не вся сцена.
- **Земля**: чуть живее ровного `#336b33` — либо лёгкая текстура/нойз, либо мягкий
  градиент/виньетка по краям поля. Дёшево, без новых ассетов если можно.
- **Туман**: `FOGMODE_EXP2`, низкая density, цвет под небо — съедает дальний край,
  добавляет глубину, поле остаётся читаемым.

## 7. Поток (где подключается в main.ts)

1. После `new Scene` и создания `ground`/walls: `resolveQuality(loadPreset())` →
   `new Environment(scene, …, cfg)`; `env.setReceiver(ground)`.
2. `makeHero()` → `env.addShadowCaster(heroBody)`.
3. `EnemyView`/`TowerView` ctor → `env.addShadowCaster(this.mesh)` (env прокинут).
4. `buildEnvironment`/`buildProp` → кастеры для базы, тайлов, solid-пропов.
5. Существующий `HemisphericLight` остаётся как fill (intensity снижается).
6. Клавиша циклит пресет → `env.applyQuality(resolveQuality(next))` + сохранить.

Игровой цикл `onBeforeRenderObservable` не трогаем (рендер-пост-фх живёт в pipeline,
тени обновляются движком автоматически).

## 8. Тестирование

- **Юнит (vitest)** — `Quality.ts`: `resolveQuality` отдаёт правильные флаги на
  каждый пресет; round-trip load/save (мок `localStorage`); неизвестный/битый
  пресет → дефолт `med`.
- **Babylon/визуал — ручной прогон в браузере** (как в существующем дизайне):
  чек-лист: тени под героем/врагами/башнями; bloom только на снарядах/вспышках, не
  на всём; туман не скрывает поле; читаемость hp-баров и цифр урона сохранилась;
  смена пресета low/med/high работает и переживает перезагрузку; FPS приемлем на med.

## 9. Риски / на что смотреть

- **Перф**: тени + SSAO дороги. Пресеты + ограничение приёмника тенью одной землёй.
  Если med тормозит — снизить shadow map / blur.
- **Динамические кастеры**: враги спавнятся по ходу — регистрировать в момент
  создания view, иначе у новых врагов не будет тени.
- **GlowLayer × SSAO**: SSAO может гасить glow (известно из доков Babylon) — на
  high проверить, что снаряды/кристалл всё ещё светятся; при конфликте исключить
  emissive-меши из SSAO.
- **applyTint в AssetManager**: красит меши flat `StandardMaterial` без specular —
  совместимо с тенями/glow; emissive у них нулевой, в bloom не попадут (ок).
- **Перепросвет**: солнце + hemi + тонмаппинг легко пересветить — подбирать
  intensity/exposure вместе, а не по отдельности.

## 10. Критерий готовности P1

Запуск в браузере: герой/враги/башни отбрасывают мягкие тени на землю; emissive
светится через bloom/glow; небо и лёгкий туман дают глубину; палитра сочная и
читаемая; пресеты low/med/high переключаются и сохраняются; `npm run typecheck` и
`npm test` зелёные; геймплей и FPS не пострадали.
